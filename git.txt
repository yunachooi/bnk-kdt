1. 깃 저장소 생성 : git init
2. 깃 저장소의 구성
	1) 작업 디렉토리 -> add
	2) 스테이지 -> commit -> 버전 생성
	3) 저장소
3. 버전을 되돌린다.
	1) revert : 되돌아간 상태에 대한 새로운 버전(커밋)을 생성.
		(안전한 방식)
	2) reset : 되돌아갈 버전 이후의 버전들은 삭제되는 방식.
		- HARD (작업 내용이 남지 않음)
		- SOFT, MIXED(작업 내용이 남아있음)
		(협업 시 사용하지 말 것)
4. branch 다루기
	1) 주 흐름에서 벗어나 새로운 흐름을 만드는 것
		- 새로운 기능을 만들 때
		- 실험적인 작업을 수행할 때
	2) 브랜치 이동 -> 체크아웃
	3) 작업이 완료된 브랜치는 메인 브랜치에 병합 -> marge
	4) 병합한 브랜치 삭제
	5) 병합 시 충돌 발생 가능성 있음.
		- 충돌 발생 시 어떤 버전의 값으로 저장할 지 선택
	6) branch 재배치
		- 브랜치가 생성된 시점을 변경 가능
5. 원격저장소와 상호작용
	1) 클론(clone) : 원격 저장소 복제하기
	2) 푸시(push) : 원격 저장소 밀어넣기
	3) 패치(fetch) : 원격 저장소를 일단 가져만 오기
	4) 풀(pull) : 원격 저장소를 가져와서 합치기(fetch, merge)
